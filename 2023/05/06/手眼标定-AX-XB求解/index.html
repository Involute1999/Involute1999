1 AX=XB问题

物体姿态估计的不确定性来自三个主要来源：

  • 相机帧中物体姿态估计不确定性；
  • 手眼标定的不确定性；
  • 机器人末端执行器定位的不确定性。

在实践中，2）的贡献最大。手眼标定是机器人视觉中的一个基本问题，它包括识别安装在机器人末端执行器上的相机和末端执行器本身之间的刚体变换。从数学上讲，这个问题可以公式化为：在AX=XB中求解X。在《手眼标定》一文中采用了Tais方法求解X，为了优化求解过程，得到相对准确的手眼矩阵，本文改用《On the Covariance of X in AX = XB》中的求解方法求解。

论文地址：[1706.03498] On the covariance of X in AX = XB (arxiv.org)

本文不仅对X的求解感兴趣，而且对从A和B的协方差中更全面地评估X的协方差感兴趣，因为现在A和B是随机扰动的变换矩阵。由于以下几个原因，这种细粒度的信息在高精度机器人应用中至关重要。

  • 在高精度制造中，不仅要知道物体的姿态，而且要保证姿态估计误差在一定的公差范围内，这一点很重要。例如，当在飞机机身上钻孔时，孔的位置公差为0.5毫米，这将被0.05的误差所违反◦ 在手眼校准中，即使假设相机帧中的物体姿态估计是完美的（如上所述）。
  • 物体姿态协方差矩阵的精确知识允许通过其他感知模式智能地细化物体姿态估计。例如，在视觉触觉传感器融合中，知道物体姿态的协方差在沿着X轴的平移中相对较大，将提示我们沿着该轴触摸物体，以最佳地降低不确定性。
  • 此外，知道 X 的协方差可以改善校准过程本身，例如，通过选择适当数量的测量以达到所需的精度水平，或选择最小化 X 协方差的适当矩阵 A 和 B。

本文将协方差的前向和后向传播方法转换到SE（3）中的不确定性框架中，先求解 $R_x$，再求解$t_x$ 。

笔者在原有代码的基础上，结合论文原理，采用manif库，使用C++重新复现了该论文。

5 问题

terminate called after throwing an instance of ‘manif::invalid_argument’

解决：https://github.com/artivis/manif/issues/171

在manif/include/manif/constant.h

static constexpr _Scalar eps_s = _Scalar(1e-15); // ~

因为四元数没有归一化，但我使用了 eigen 提供的四元数归一化函数。这个问题是1e-15 不适合float32 值，1e-6 会更适合它。
